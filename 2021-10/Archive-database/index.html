不可视境界线最后变动于：2022年1月5日 晚上

这篇主要是记录一下用过的语句, 毕竟文档是在太多只能用到了再去记下来

语法

  • alter语句
    • 这个吧直接查文档就行
  • DATE DATETIME ……
    • 三种形式: 有引号有分隔符 | 有引号无分隔符 | 纯数字
    • STR_TO_DATE()就如字面意思
  • PRIMARY KEY()定义联合主键, 联合主键就是由1个以上的字段构成的唯一标识.
  • 查看表的约束和其他信息
    • select * from information_schema.table_constraints; 查询所有表的约束
    • show create tabel [table_name]
  • mysql清屏: system cls 只在cmd powershell 和 Cmd Client中起作用
  • WITH关键字生成一个临时表
  • GROUP BY后面的HAVING字句如果按照SQL的标准只能引用GROUP BY中的字段名称, 但是MySQL(较新版)做了一个拓展: HAVING字句中可以包含聚集函数、SELECT中的字段名称、外层子查询中的字段名
  • MySQL不支持EXCEPT和INTERCECT
  • 内置函数: IFNULL()
  • WITH CLAUSE
  • HAVING 字句在投影运算之前
  • MySQL变量不需要事先定义, 如果不需要用了直接set @n = null
  • 自然连接时后加进来的表的外键如果在待连接表之中, 则选择条件应为所有外键值对都相等
  • NATURAL JOIN 不能乱用, 它见到相同的字段名称就拿去匹配, 会出现意料之外的Empty Set
    • 破案了, 每张表的字段名称最好特别一点, 不要只是name这种通用名称, 最好是EmpName之类的
  • COALESCE() 返回第一个非空值
  • VALUES statement(?
  • IF() and compound statement syntax

报错信息

  • 1822 - Failed to add the foreign key constraint. Missing index for constraint ‘booking_ibfk_3’ in the referenced table ‘guest’
    • 原因：表A如果没有设置主键，那么表A里面的字段是不能作为表B的外键的。
  • 3813 - Column check constraint ‘customer_details_chk_2’ references other column
    • 硬搬书上的什么check(value in(‘sldfkj’, ‘ksdjf’))导致错误, 应该把value改成字段名称
  • 1248 - Every derived table must have its own alias
    • 用进行嵌套查询的时候，子查询执行出来的的结果是需要作为一个派生表来给上一级进行查询使用的，所以子查询的结果必须要有一个别名
